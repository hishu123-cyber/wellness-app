> **版本：v2.0 · 2026年6月**

## 零、这是一个什么样的项目？

不是又一个告诉你"多吃蔬菜多运动"的健康APP。

**它是你的"体质翻译官"。**

你做了体检测试，拿到一堆指标数据，但不知道这意味着什么。
你妈转给你一篇养生文章，但不知道适不适合你。
你感觉自己湿气重，但不知道该吃什么、不该吃什么。

**体质养生·大健康** 要做的事就是——
花1分钟让你知道自己是什么体质，然后AI告诉你：你该吃什么、不该吃什么、做什么运动、怎么过四季。

然后，你点一下，食材直接送到家。

## 一、市场机会：五个让人睡不着觉的事实

### 事实1：14亿人里，真正"健康"的不到5%

截至2025年，中国符合WHO健康定义的人口比例约为 5%，其余 95% 处于疾病或亚健康状态：

- 高血压患者：**3.3亿+**
- 糖尿病患者：**1.4亿+**
- 超重/肥胖人群：**超6亿**
- 高血脂：**超4亿**
- 自认"亚健康"的职场人：**76%**

这不是一组数字。这是4亿中年人每天下午头晕、8000万老年人每天数着药片、3亿女性每年花3000亿在减肥上的真实生活。

### 事实2：养生信息泛滥，但几乎没有可信的"个性化"方案

现状：
- 百度搜养生 → 广告和软文
- 小红书搜食谱 → 网红推荐，不一定适合你的体质
- 微信家族群 → 父母转的"震惊体"，你敢信？
- 丁香医生 → 偏医疗诊断，不解决"吃什么"

**用户需要的不是更多信息，是一个懂自己的专属方案。**

### 事实3：消费升级最先升级的，是"怕死"

中国健康食品市场过去5年年均增长 15%+：
- 功能食品：2025年超 **3000亿**
- 有机食品：年增速 **20%+**
- 健康管理APP下载量：年增 **40%+**
- "花钱买健康"已经从概念变成刚需

### 事实4：老龄化不是未来，是现在

- 60岁以上人口：**3亿+**（2025年）
- 其中空巢老人：**超1.3亿**
- 子女不在身边，老人不知道该怎么吃、该买什么
- 他们不信任网络，但极度信任"专业人士"

这是一个**子女付费、父母享用**的完美消费链路。

### 事实5：你不需要教育市场，市场已经被教育好了

- 2016-2025这十年，**Keep、薄荷、下厨房、叮咚买菜**已经把用户的习惯养成了
- 用户会为健康内容付费、会上买菜APP、会跟着食谱做饭
- 缺的是一环：**连接"我是谁"和"我该吃什么"**

## 二、我们的解决方案

### 核心逻辑：三步闭环

```
      第1步                   第2步                   第3步
 ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
 │   了解自己    │  →  │   获得方案    │  →  │   立即行动    │
 │              │      │              │      │              │
 │ 1分钟体质测评 │      │ AI配食谱     │      │ 一键购食材   │
 │ 9种体质识别   │      │ AI运动建议   │      │ 订阅配送     │
 │ 健康日记追踪  │      │ 节气养生提醒  │      │ 24h送到家    │
 │ 趋势图可视化  │      │ 每周健康报告  │      │              │
 └──────────────┘      └──────────────┘      └──────────────┘

    壁垒：越用越懂你         服务      交易
    数据积累形成护城河
```

### 已完成的全部功能（不是PPT，是跑在服务器上的代码）

**体质测评引擎**
- 45道专业问答 → 9种体质（气虚/阳虚/阴虚/痰湿/湿热/血瘀/气郁/特禀/平和）精准识别
- 每次测评记录都保存，形成体质变化趋势图

**AI健康助手（规则引擎版 + AI API版双模式）**
- 9种体质全覆盖的饮食/运动/穴位/四季/情绪/注意事项
- 按性别年龄定制个性化建议
- 一周食谱：按体质定制每日三餐+茶饮（168种组合）
- 7天挑战：每日任务计划
- 症状自查：输入症状→分析体质倾向
- 预留 DeepSeek / OpenAI / SiliconFlow API，接入后从规则模式切换为AI大模型模式

**健康追踪**
- 日记：睡眠/运动/饮食/饮水/心情 每日记录
- 7天趋势图：Canvas 动态生成
- 健康分算法：每日综合评分（0-100）
- 每周健康报告

**养生内容**
- 24节气养生：当前节气+全部节气，点开有饮食/运动/穴位建议
- 养生文章库：分类筛选阅读
- 食疗菜谱：15个食谱，按体质/分类筛选

**商城（MVP版）**
- 商品浏览：分类/搜索/排序
- 购物车：增删改查
- 订单系统：创建/查看
- 营养师展示+预约（框架已搭好）

**付费能力**
- VIP会员体系
- 积分系统（签到/打卡/兑换）
- 支付接口（微信/支付宝 对接就绪）

**用户体验**
- 适老化设计：大字号、高对比度、毛玻璃效果
- 深色模式：系统检测+手动切换
- PWA：可安装到手机桌面，离线可用
- 骨架屏加载：减少白屏等待感

**技术基石**
- API：Express 5 + JWT 认证，20+ 端点
- 测试：28项集成测试全部通过
- 部署：阿里云 ECS（Ubuntu 2C2G）+ Docker + Nginx
- 数据库：sql.js（随时可迁移至 MySQL/PostgreSQL）

### 定价策略

> "测评完就能直接买适合体质的食材"——这就是我们的飞轮。

>
> **用户不需要第100个菜谱APP，他们需要一个能回答"我该吃什么"并帮他们买到的工具。**

## 六、为什么现在投

### 1. 产品已经在了，不是画饼
很多创业项目在BP阶段就要钱。
我们是从PRD写到了代码跑在云服务器上的。

你可以打开手机，填写体质测评，拿到报告。
你可以浏览食谱，加购物车，下一个订单。
所有核心功能都是活的——不是示意图。

### 2. 大健康赛道最大的红利还没吃完
健康管理APP市场年增速20%。
4亿中年人开始认真养生。
3亿老年人第一次学会用智能手机下单买菜。
**风口还在，但窗口在收窄。**

### 3. 这个模式的壁垒是数据
每多一个用户测评，体质数据库多一份样本。
每多一个用户记日记，健康趋势模型精准一分。
越多人用，推荐越准。
越多人用，商城SKU越匹配。
**先发优势在这里不是空话——后入者要花两倍的钱才能追上。**

### 4. 多种变现路径，降低单点风险
- 有人用AI → 转会员
- 有人买东西 → 转商城
- 有人需要营养师 → 转服务
- 有企业想买员工福利 → 转B端

产品只有一个，变现出口有四个。
任何一个跑通，项目就活了。
